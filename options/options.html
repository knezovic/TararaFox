Watched tabs

Each entry opens one tab. Network responses inside that tab are matched by URL pattern and content type, then forwarded to the API endpoint.

On Tab URL & URL patterns Content types Refresh (s) Scroll Active

Patterns: one per line or comma-separated, * matches anything (e.g. https://example.com/api/*). Leave empty to match every request in the tab.

Select WebSocket to also capture incoming WebSocket messages for the tab. Outgoing frames are intentionally left untouched so the page's live feed is never disturbed. Messages are matched by the same patterns against the socket's wss:// URL or its https:// equivalent, so either scheme works (leave empty to capture every message). Messages received inside Web Workers cannot be captured.
